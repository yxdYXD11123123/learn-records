# 交换方式

# 电路交换

电话问世后，人们发现，如果想要让所有电话都连接起来是不可能的，如下图：

于是，人们发明了**电话交换机**：用一个电话交换机把所有电话连接起来。

  • 电话交换机接通电话的方式称为电路交换

  • 从通信资源的分配角度来看，交换（Switching）就是按照某种方式动态地分配传输线路的资源

    看一个电路交换的示意图：

  • 电路交换的三个步骤：

    • 建立连接（分配通信资源）
    • 通话（一直占用通信资源）
    • 释放连接（归还通信资源）

那么，如果使用电路交换来传输计算机数据，是否可行？

  • 虽然可以实现传输数据，但是线路的传输效率会很低，因为计算机数据是突发式地出现在传输线路上的，会造成所占用的通信资源暂时未被利用。因此计算机数据通常采用分组交换而不是电路交换。

# 分组交换（packet Switching）

假设H6 主机要给H2 主机用户发送一个消息：

  • 首先，主机 H6 将消息打包为一个报文
  • 将报文划分成一个个更小的等长的数据段
  • 在每个数据段前面加上一些由必要的控制信息组成的首部，就构成了一个个分组（也可简称为“包”，首部也可称为“包头”）
  • 将分组们发给分组交换器（路由器）
  • 分组交换机收到一个分组后：
    • 先将分组暂时存储下来，
    • 再检查分组首部
    • 按照首部中的目的地地址进行查表转发
    • 找到合适的转发接口
    • 通过该接口将分组转发给下一个分组交换机
  • 最终到达主机 H2
  • 主机 H2 收到分组后，去掉所有分组的首部，将各数据段组合还原出原始报文

在上述分组交换过程中，

  • 发送方任务：构造分组、发送分组
  • 分组交换机（路由器）任务：缓存分组、转发分组（存储转发）
  • 接收方任务：接收分组、还原报文

# 报文交换

报文交换与分组交换类似，也是通过报文交换机完成存储转发，只是报文交换的报文，没有大小限制（因此需要报文交换机有较大的存储能力）

主要是用于早期的电报通信，现在基本被分组交换所取代。

# 对比：电路交换、报文交换、分组交换

# 电路交换优缺点：

  • 优点：
    1. 通信时延小
    2. 有序传播
    3. 没有冲突（有专用通道）
    4. 适用范围广
    5. 实时性强
    6. 控制简单
  • 缺点：
    1. 建立连接时间长
    2. 线路独占，使用效率极低
    3. 灵活性差
    4. 难以规格化、规范化

# 报文交换优缺点：

  • 优点：
    1. 无需建立连接
    2. 动态分配线路
    3. 提高线路可靠性
    4. 提高线路利用率
    5. 提供多目标服务
  • 缺点：
    1. 引起了转发时延（报文在结点交换机上要经过存储转发的过程）
    2. 需要较大的存储缓存空间（报文不限制大小）
    3. 需要传输额外的信息量（例如：目标地址、发送地址等控制信息）

# 分组交换优缺点：

  • 优点：

    1. 无需建立连接
    2. 线路利用率高
    3. 简化了存储管理
    4. 加速传输（每个分组是独立转发的，在一定程度上是并行发送的）
    5. 减少出错概率和重发数据量（分组小块传输，失败概率小）

  • 缺点：

    1. 引起了转发时延

    2. 需要传输额外的信息量（每个分组块都要带上完整的控制信息）

    3. 对于数据包服务，存在失序、丢失或重复分组的问题；

      对于虚电路服务，存在呼叫建立、数据传输和虚电路释放三个过程
