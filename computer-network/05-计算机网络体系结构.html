# 计算机网络体系结构

# 常见的计算机网络体系结构

为了使不同体系结构的计算机网络都能互联，国际标准化组织于 1977 年成立专门机构研究该问题。

不久，就提出了一个试图使各种计算机在世界范围内互联成网的标准框架，也就是著名的“开放系统互连参考模型”，简称 OSI

# OSI 体系结构

该模型是一个七层协议的体系结构

  • 该模型是法律上的国际标准

# TCP/IP 体系结构

虽然整套的 OSI 国际标准都已经制订出来了，但这时因特网已抢先在全世界覆盖了相当大的范围，

因特网在 1983 年开始使用 TCP/IP 协议族，并逐步演变成 TCP/IP 参考模型，该模型是一个四层协议的体系结构

  • 该模型是事实上的国际标准

OSI 标准失败的原因可归纳为：

  • OSI 的专家们缺乏实际经验，在完成 OSI 标准时，没有商业驱动力
  • OSI 的协议是实现起来过分复杂，而且运行效率很低
  • OSI 标准的制定周期太长，因而使得按 OSI 标准生产的设备无法及时进入市场
  • OSI 的层次划分也不太合理，有些功能在多个层次中重复出现

对比 OSI 和 TCP/IP 体系结构，可以看出：

  • TCP/IP 相当于将 OSI 体系结构的物理层和数据链路层合并为了**网络接口层**。
  • 去掉了会话层和表示层
  • 由于 TCP/IP 在网络层使用的协议是 IP 协议，IP 协议的中文翻译为网际协议，因此称为**网际层**。

在用户主机的操作系统中，通常都带有符合 TCP/IP 体系结构标准的 TCP/IP 协议族；

而用于网络互连的路由器中，也带有符合 TCP/IP 体系结构标准的 TCP/IP 协议族，只不过路由器一般只需包含网络接口层和网际层。

# TCP/IP 各层协议

  • TCP/IP 体系结构的网络接口层并没有规定什么具体的内容，这样做的目的是可以互连全世界不同的网络接口，例如：有线的以太网接口、无线局域网的 WiFi 接口，而不限定仅使用一种或几种网络接口

    因此，本质上 TCP/IP 体系结构只有上面的三层

  • IP 协议是 TCP/IP 体系结构网际层的核心协议

    • IP 协议可以将不同的网络接口互连，并向其上的TCP 协议和UDP 协议提供网络互连服务

  • TCP和UDP是 TCP/IP 体系结构运输层的两个重要协议

    • TCP 协议在享受 IP 协议提供的网络互连服务的基础上，可向应用层的相应协议提供可靠传输的服务。
    • UDP 协议在享受 IP 协议提供的网络互连服务的基础上，可向应用层的相应协议提供不可靠传输的服务。

  • TCP/IP 体系结构的应用层包含了大量的应用层协议，例如：HTTP、SMTP、DNS、RTP 等

IP 协议作为 TCP/IP 体系结构中的核心协议，

  • 一方面负责互连不同的网络接口（IP over everything）
  • 另一方面，为各种网络应用提供服务（Everything over IP）

IP 协议和 TCP 协议是 TCP/IP 协议中非常重要的两个协议，因此用这两个协议来表示整个协议大家族，常称为**TCP/IP 协议族**

# 原理体系结构

因为 TCP/IP 的网络接口层没有规定什么具体内容，对于我们学习计算机网络的完整体系而言，会缺少一部分内容。

因此，在学习计算机网络原理时，往往会采取折中的办法：综合 OSI 和 TCP/IP 的优点，采用一种五层协议的原理体系结构，将网络接口层分出“数据链路层”、“物理层”，这样更有利于学习完整的计算机网络体系

# 计算机网络体系结构分层的必要性

  • 计算机网络是个非常复杂的系统，早在最初的 ARPANET 设计时，就提出了分层的设计理念
  • “分层”可将庞大而复杂的问题，转化为若干较小的局部问题，而这些较小的局部问题就比较易于研究和处理

# 最简单的情况：

两台计算机通过一条网线连接起来，

对于这种最简单的情况，我们需要考虑以下问题：

  1. 采用怎样的传输媒体（介质）：双绞线网线

  2. 在计算机上采用怎样的物理接口来连接传输媒体：RJ45 以太网接口

  3. 使用怎样的信号表示比特 0 和 1

    • 使用方波信号：

      低电平表示 0，高电平表示 1

确定了这些问题，两台计算机就可以通过信号来传输比特 0 或 1 了

我们可以将这些问题划归到**物理层**

# 注意：
  • 严格来说，传输媒体并不属于物理层，并不包含在体系结构之中
  • 计算机网络中传输的信号，并不是举例的方波信号（暂时为了便于理解）

# 多台计算机网络

实用的计算机网络，一般都由多台主机构成

例如上图的总线型网络

假设，我们已经解决了物理层的问题，也就是主机之间可以传输 0 和 1 了，我们来看看，这样的总线型网络上，还面临什么需要解决的问题？

# 主机 A 要给主机 C 传输数据

  • 问题一：主机 C 如何知道数据是发送给自己的，自己要接收？主机 B、D、E 又如何知道该数据并不是发送给自己的，自己应该拒绝呢？

    该问题就是 如何 标识网络中的各主机问题（主机编址问题，例如MAC 地址）

    网卡上的 MAC 地址，就是主机在网络中的地址，主机在发送数据时，应该给数据附加上目的地址，当其他主机收到后，根据目的地址和自身地址是否相同，决定是否接受数据。

  • 问题二：如何从信号所表示的一连串比特流中区分出地址和数据？

    也就是需要解决**分组的封装格式**问题

# 主机 B 和主机 D 同时发送数据

B、D 主机同时发送数据，必然会造成信号碰撞

  • 问题一：如何协调各主机征用总线？

上述问题可以划分到**数据链路层**

只要解决了物理层和数据链路层各自面临的问题，我们就可以实现分组在一个网络上传输了。

# 多个网络之间传输数据

  • 问题一：如何标识各网络以及网络中的各主机（网络和主机共同编址的问题，例如 IP 地址）
  • 问题二：路由器如何转发分组，如何进行路由选择

上述问题可以划分到**网络层**

解决了上述问题，我们就可以实现分组在网络间传输的问题

# 主机中运行着两个与网络通信相关的应用进程

主机收到来自其他主机发送的分组，那应该交给浏览器进程处理，还是 QQ 进程处理？

  • 问题一：如何标识与网络通信相关的应用进程？解决进程之间基于网络通信的问题
  • 问题二：如何分组传输过程产生了丢包问题，出现传输错误，应该如何处理？

上述问题可以划归到**运输层**

此时，我们就可以实现主机进程间基于网络的通信

在此基础上，只要制订各种应用协议，并按照协议标准编写相应的应用程序，通过应用程序之间的交互来完成特定的网络应用。

例如，支持万维网应用的 HTTP 协议、支持电子邮件的 SMTP 协议、支持文件传送的 FTP 协议，

我们将这些问题全部划分到**应用层**

# 计算机网络体系结构分层思想举例

当我们访问一个网站时，

主机和 Web 服务器之间基于网络的通信，实际上是主机浏览器的应用进程与Web 服务器中的 Web 服务器应用进程之间基于网络的通信

那么，在此过程中，体系结构中的各层的作用是什么的？

从主机端按体系结构自顶向下的顺序来看，

  • 应用层按 HTTP 协议的规定，构建一个HTTP 请求报文

应用层将 HTTP 请求报文交付给运输层处理，

  • 运输层给 HTTP 请求报文添加一个 TCP 首部，使之成为TCP 报文段，这是 TCP 报文端的首部格式
    • TCP 报文端首部的作用，主要是为了区分应用进程以及实现可靠传输

运输层将 TCP 报文段交付给网络层处理

  • 网络层给 TCP 报文段添加一个 IP 首部，使之成为IP 数据报，这是 IP 数据报的首部格式
    • IP 数据报首部格式的作用主要是为了使 IP 数据报可以在互联网上传输，被路由器转发

网络层将 IP 数据报交给数据链路层处理

  • 数据链路层将帧交付给物理层（同上）
  • 物理层将帧看作是比特流，给比特流添加前导码（同上）

信号通过传输媒体到达 Web 服务器，我们来看Web 服务器的处理过程：

  • 物理层将信号处理成比特流，去掉前导码，将帧交付给数据链路层
  • 数据链路层将帧的首部和尾部去掉后，将其交付给网络层
  • 网络层将 IP 数据报的首部去掉后，将其交给运输层，这实际上交付的是TCP 报文段
  • 运输层将 TCP 的报文段去掉后，将其交付给应用层，这实际上交付的是HTTP 请求报文
  • 应用层对 HTTP 请求报文进行解析，然后给主机发回 HTTP 响应报文（发回过程不再复述）

# 计算机网络体系结构中的专用术语

首先，我们将专用术语中最具有代表性的三个作为分类名称：实体、协议、服务

# 实体

  • 实体：指任何可发送或接收信息的硬件或软件进程

  • 对等实体：收发双方相同层次中的实体

# 协议

  • 协议：控制两个对等实体进行逻辑通信的规则的集合

    • 之所以称为逻辑通信，是因为这种通信其实并不存在，它只是我们假设出来的一种通信，目的在于方便我们单独研究体系结构某一层时，不用考虑其他层

  • 协议的三要素：

    • 语法：定义所交换信息的格式

      例如 IP 数据报：

    • 语义：定义通信双方所要完成的操作

      例如：浏览器和服务器在接收到 HTTP 请求/响应后，要进行渲染页面或响应处理，也就是通信双方收到分组后，要完成怎样的操作。

    • 同步：定义通信双方的时序关系

      例如 TCP 采用三次握手建立连接的过程：

      • 要想进行运输层 TCP 实体间的逻辑通信，首先必须建立连接
      • 从连接建立的过程可以看出 TCP 客户端和 TCP 服务端之间的时序关系，以及各自的状态转换
      • 只有双方建立连接后，才能进行 TCP 数据传输

# 服务

在协议的控制下，两个对等实体间的逻辑通信使得本层能够向上一层提供服务

要实现本层协议，还需要下面一层协议所提供的服务

  • 服务访问点：在同一系统中相邻两层的实体交换信息的逻辑接口，用于区分不同的服务类型

    • 数据链路层的服务访问点为帧的“类型”字段
    • 网络层的服务访问点为 IP 数据报首部中的“协议字段”
    • 运输层的服务访问点为“端口号”

  • 服务原语：

    上层要使用下层的服务，必须通过与下层交换一些命令，这些命令称为服务原语

在计算机网络体系结构中，对通信双方所交互的数据包有专门的术语

  • 协议数据单元 PDU：对等层次之间传送的数据包

    例如，物理层的 PDU 就是比特流；数据链路层的 PDU 就是帧；网络层的 PDU 就是 IP 数据报/分组

  • 服务数据单元 SDU：同一系统内，层与层之间交换的数据包

  • 多个 SDU 可以合成为一个 PDU，一个 SDU 也可划分为多个 PDU
