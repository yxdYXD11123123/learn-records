# 以太网交换机的生成树协议 STP

如何提高以太网的可靠性？

例如，在上图以太网结构中，如果交换机A与B之间的链路出现了故障，

则交换机B上连接的所有主机，无法与交换机A和交换机C上的所有主机进行通信

**解决方案：**通过添加冗余链路可以提高以太网的可靠性

给交换机B和C之间添加一条冗余链路，即使交换机A和B之间的链路出现故障，整个网络还是连通的

但是，冗余链路也会带来负面效应——形成网络环路

网络环路会带来以下问题：

  • 广播风暴：当某主机发送广播帧时，广播帧会在以太网交换机之间循环转发

    大量消耗网络资源，使得网络无法正常转发其他数据包

  • 主机会收到重复的广播帧

    大量消耗主机资源

  • 交换机的帧交换表震荡（漂移）

    在交换机不断重复接收该广播帧时，帧交换表中的登记信息（对应接口信息）被不断修改

# 生成树协议STP

为了可以在增加冗余链来提高网络可靠性的同时，又避免网络环路带来的各种问题

以太网交换机使用生成树协议STP（Spanning Tree Protocol），可以在增加冗余链路来提高网络可靠性的同时又避免网络环境带来的各种问题

  • 不论交换机之间采用怎样的物理连接，交换机都能够自动计算并构建一个逻辑上没有环路的网络，其逻辑拓扑结构必须是树形的（无逻辑环路）

    如上图所示，交换机对自己的一些**接口进行阻塞，**这样就会形成一个逻辑上没有环路的网络

  • 最终生成的树型逻辑拓扑需要确保连通整个网络

  • 当首次连接交换机或网络物理拓扑发送变化时，交换机都将进行生成树的重新计算。
