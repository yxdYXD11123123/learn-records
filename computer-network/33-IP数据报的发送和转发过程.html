# IP数据报的发送和转发过程

  • IP数据报的发送和转发过程包含以下两个部分：

    • 主机发送IP数据报

      判断目的主机是否与自己在同一个网络

      • 若在同一个网络，则属于直接交付
      • 若不在同一个网络，则属于间接交付，传输给主机所在网络的默认网关（某路由器接口），由默认网关的路由器帮忙转发

    • 路由器转发IP数据报

      1. 首先检查IP数据报首部是否出错：

        • 若出错，则直接丢弃该IP数据报并通告源主机
        • 若没有出错，则进行转发

      2. 根据IP数据报的目的地址在路由表中查找匹配的条目

        根据目的地址在路由表中通过每条记录的地址掩码去计算找出目的网络地址的对应接口，通过该接口转发出去，到达目的网络

        • 若找到匹配的条目，则转发给条目中指示的下一跳
        • 若找不到，则丢弃该IP数据报并通告源主机

    为了将重点放在TCP/IP协议栈的网际层发送和转发IP数据报的过程上，在之后的举例中，我们忽略使用ARP协议来获取目的主机或路由器接口的MAC地址的过程以及以太网交换机自学习和转发帧的过程

# 源主机如何知道目的主机是否与自己在同一网络中呢？

假设，主机C要给主机F发送数据报

  • 将主机C的IP地址与子网掩码相与，就可以得到C的网络地址
  • 将目的主机F的IP地址与子网掩码相与，就可以得到目的网络地址
  • 两个网络地址对比，不相等，则不在同一个网络中，就属于间接交付
  • 主机C需要将IP数据报传输给路由器，由路由器将IP数据报转发给主机F

# 主机C如何知道应该把IP数据报交给哪个路由器进行转发呢？

用户为了让本网络中的主机可以和其他网络的主机通信，就必须给其指定本网络中的一个路由器，由该路由器帮忙进行转发，所指定的路由器，也被称为默认网关。
