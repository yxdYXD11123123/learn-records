# 点对点协议PPP

  • 点对点协议PPP（Point-to-Protocol）是目前使用最广泛的点对点数据链路层协议

    • 用户计算机与ISP服务商通信时，所使用的数据链路层协议通常就是PPP协议

    • 1999年公布的在以太网上运行的PPP协议（PPP over Ethernet）简称PPPoE，它使得ISP可以通过DSL、电路调制解调器、以太网等宽带接入技术，以以太网接口的形式为用户提供接入服务

      • 点对点协议PPP，也广泛应用于广域网络路由器之间的专用线路

  • PPP协议为在点对点链路传输各种协议数据报提供了一个标准方法，主要由以下三部分构成：

    • 对各种协议数据报的封装方法（封装成帧）
    • 链路控制协议LCP：用于建立、配置以及测试数据链路的连接
    • 一套网络控制协议NCPs：其中的每一个协议支持不同的网络层协议（IP、IPX、AppleTalk）

# 帧格式

标志（Flag）字段：PPP帧的定界符，取值为0x7E

地址（Address）字段：取值为0xFF，预留（目前没有什么用）

控制（Control）字段：取值为0x03，预留（目前没有什么用）

协议（Protocol）字段：指明帧的数据部分送交哪个协议处理

帧检验序列（Frame Check Sequence）字段：CRC计算出的校验位，用于检验是否存在误码

# 透明传输

当PPP帧的数据部分出现帧首或帧尾中的标志字段时，如果不采取措施，会造成接收方对PPP帧是否结束的误判。

# 方法

PPP实现透明传输的方法取决于所使用的链路类型

  • 字节填充法

    • 发送方的处理
      • 每出现一个7E（PPP帧的定界符）字节，转变成2字节序列（7D,5E）
      • 每出现一个7C（转义字符）字节，转变成2字节序列（7D，5D）
      • 每出现的一个ASCII码控制字符（数值小于0x20的字符），则在该字符前面插入一个7D字节，同时将该字符的编码加上0x20。
    • 接收方的处理：
      • 进行反变换即可恢复出原来的帧的数据部分

  • 比特填充法

    帧定界符位：0111110

    • 发送方的处理：
      • 对帧的数据部分进行扫描（一般由硬件实现）。只要发现5个连续的比特1，则立即填充1个比特0
    • 接收方的处理
      • 对帧的数据部分进行扫描（一般由硬件实现），只要发现5个连续的比特1，则把其后的1个比特0删除

# 差错检测

接收方每收到一个PPP帧，就进行CRC循环冗余校验，若校验正确，就收下这个帧；反之，就丢弃这个帧，使用PPP的数据链路层向上提供不可靠传输服务。
